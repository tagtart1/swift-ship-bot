# Project guidance

## grammY

When working with grammY:

- Check the installed grammY version before implementing anything.
- Consult the current official documentation at https://grammy.dev/.
- Use the official API reference at https://grammy.dev/ref/.
- Confirm APIs against the installed TypeScript declarations.
- Prefer official grammY documentation over remembered examples or third-party tutorials.
- Cite the relevant documentation when explaining grammY behavior.

## NestJS

When working with NestJS:

- Check the installed NestJS and Nest CLI versions before implementing anything.
- Consult the current official documentation at https://docs.nestjs.com/.
- Use the official NestJS API reference when exact framework APIs or types matter.
- Confirm APIs against the installed TypeScript declarations.
- Prefer established NestJS patterns for modules, providers, dependency injection, configuration, lifecycle management, validation, testing, and application structure.
- When adding or changing an integration such as Prisma, first consult the relevant NestJS guide or recipe, then consult that integration's current official documentation and installed TypeScript declarations.
- Keep third-party integrations behind clear NestJS module and provider boundaries when appropriate.
- Prefer official NestJS documentation and official integration documentation over remembered examples, outdated tutorials, or third-party boilerplates.
- Cite the relevant documentation when explaining NestJS behavior or recommending an integration approach.
